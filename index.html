Chat UX – Rol × nieuw bericht

Notificaties gelden alleen voor berichten tussen admin en adviseur (en andersom). Als een admin een bericht stuurt, krijgt de andere admin geen notificatie.

Altijd zichtbaar: Het dossier is te allen tijde boven in de werkvoorraad zichtbaar, ook al is er een filter actief.

Niet altijd zichtbaar: Het dossier is enkel zichtbaar als er geen filter actief is of als het dossier voldoet aan de filtercriteria.

Legenda migratie: het rode bolletje is de vervanging van de zwarte kleur uit de oude implementatie. De pin is de vervanging van de blauwe kleur.

Rol × nieuw bericht (matrix)

Matrix met alle toegestane combinaties voor Admin A en Admin B. Het rode bolletje verschijnt alleen bij altijd zichtbaar. Als een admin het bericht leest, verdwijnt het bolletje voor die admin.

Scenario Admin A Admin B
A niet altijd zichtbaar – B niet altijd zichtbaar
A altijd zichtbaar – B altijd zichtbaar
A altijd zichtbaar – B altijd zichtbaar (nieuw bericht van adviseur)
A altijd zichtbaar – B altijd zichtbaar (na lezen door A)
A altijd zichtbaar – B altijd zichtbaar (na lezen door B)
A altijd zichtbaar – B altijd zichtbaar (Admin A stuurt bericht)
A altijd zichtbaar – B altijd zichtbaar (Admin B stuurt bericht)

Flow tabel (uit matrix)

Onderstaande flow vertaalt de matrix naar concrete situaties in de praktijk. Zodra Admin A of Admin B het bericht leest, verdwijnt het rode bolletje voor die admin. Bij een admin-bericht krijgt de andere admin geen notificatie.

Stap Omschrijving Admin A Admin B
1 Bij een nieuw binnenkomend dossier of dossier uit archief staan dit bij beide admins op niet altijd zichtbaar.
2 Er komt een nieuw bericht van de adviseur binnen; het dossier wordt dan bij beide admins altijd zichtbaar.
3 Admin A leest het bericht; het rode bolletje verdwijnt bij Admin A.
4 Admin B leest daarna ook; het rode bolletje is nu bij beide weg.
5 Admin A stuurt een bericht naar de adviseur; Admin B krijgt geen notificatie.
6 Admin A wisselt naar niet altijd zichtbaar.
7 Er komt een bericht van de adviseur binnen; beide admins gaan naar altijd zichtbaar met rood bolletje.

3️⃣ Pin tabel

Een pin is altijd persoonlijk per admin en staat volledig los van notificaties en chats.

Status Weergave Betekenis
Actief Reminder: er staat nog een actie open op de chat of het dossier.
Pin uit Geen reminder op de chat of het dossier.

4️⃣ Auto pin (per admin)

Auto pin is een persoonlijke instelling per admin. Als auto pin aan staat, wordt het dossier bij een nieuw bericht automatisch gepind voor die admin. Dit werkt per admin afzonderlijk.

Stap Situatie Admin A Admin B
1 Nieuw bericht, auto pin A = aan, auto pin B = uit
2 Nieuw bericht, auto pin A = aan, auto pin B = aan
3 Nieuw bericht, auto pin A = uit, auto pin B = uit
